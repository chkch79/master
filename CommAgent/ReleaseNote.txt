# Upgrade notes for version v8.0.0.1:

1. There are a few table structure changes for facing audit module.
	- With new version 8.0.0.0, facing audit result is captured by brand instead of sub brand. 
	- Hence, existing facing captured under sub-brand will be aggregated under their brand. The affected tables are TXN_MERC_FACINGAUDITPRD and RPT_MERC_FACINGAUDIT.
	- TXN_MERC_FACINGAUDITPRD table will have zero total facing of category (TTL_FVALUE) and zero facing percentage (VALUE) for existing facing whereas
	- RPT_MERC_FACINGAUDIT table will sum up the total existing facing under each category as the total facing of category (TTL_FVALUE) and calculated out a facing percentage (VALUE)
	- Existing facing captured from HHT will not able to submit to server. Project team need to manually handle the migration for the failed txn in CA.
	- These changes will be reflected in below CA releases: 8.0.0.0.20151116
